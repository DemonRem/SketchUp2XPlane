X-Plane Plugin für SketchUp

Übersicht

SketchUp ist ein freier (kostenfreier) 3D Objekt Editor für Windows und MacOS.

Dieses SketchUp Plugin importiert und exportiert Modelle im X-Plane v8/v9/v10 .obj Format.

Installation

Kopiere die Dateien aus dem .zip Archive in deinen SketchUp Plugins Ordner:

  • Windows: C:\Program Files\Google\Google SketchUp n\Plugins
  • Mac OS: /Library/Application Support/Google SketchUp n/SketchUp/plugins

SketchUp staren oder neu starten.

Du kannst nun die .zip Datei, die du herunter geladen hast, löschen. Die Hilfe-Datei, die du mit herunter geladen hast, kannst du unter Hilfe → X-Plane in SketchUp aufrufen.

X-Plane-Szenerie-Objekte herstellen

Bevor du anfängst, kannst du ein Satellitenfoto von der Gegend zu bekommen, an an der du arbeiten willst:

  • Wähle Fenster → Modellinformationen → Geostandort → Ort hinzufügen…

Stelle dein Modell in SketchUp aus Rechtecken und Kreisen. Sichere die SketchUp .skp Datei in dem Aircraft- oder Scenery-Ordner, den du für X-Plane .obj Dateien vorgesehn hast.

In einem Bildbearbeitungsprogramm musst du eine einzige Textur-Datei wie eine "collage" von allen Texturen herstellen, die du in deinem Modell brauchst. Höhe und Breite deiner Textur Datei müssen eine Zweier-Potenz haben (z.B 128, 256, 512, 1024, 2048), aber sie muss kein Quadrat sein. Sichere die Textur-Datei im 32bit oder 24bit PNG Format (d.h mit oder ohne "Alpha"-Kanal) in dem Aircraft- oder Scenery-Ordner, den du für deine X-Plane .obj Datei vorgesehen hast.

Benutze das Farbeimer-Funktion, deine Textur-Datei auf jede Fläche deines Modells zu übertragen. Benutze Kontextmenü → Textur → Position, um den korrekten Ausschnitt aus deiner Textur-Datei auf jeder Fläche zu positionieren.

Klicke hier für ein Tutorial, wie man für ein Textur auf einer gekrümmten Oberfläche positionieren willst. Oder, du kannst zuerst Ansicht → Ausgeblendete Geometrie auswählen. Dann selektiere eine einzelne Fläche auf dieser Oberfläche und positioniere nacheinander Textur auf jede Einzelfläche.

Wenn du möchtest, dass beide Seiten einer Fläche in X-Plane dargestellt werden, dann benutze das Farbeimer-Funktion zweimal; je einmal für jede Seite der Fläche. Du kannst Fenster → Elementinformationen benutzen, um zu sehen, welche Textur(en) gerade den zwei Seiten der ausgewählten Fläche zugewiesen sind.

Benutze Tools → Highlight Untextured als Hilfe beim Texturieren. Das ändert nichts daran, wie dein Modell in X-Plane erscheinen wird, aber es macht irgendwelche Probleme beim Texturieren deutlich:

  1. Flächen ohne Textur sind rot gefärbt. Das hilft dir, diese Flächen zu erkennen.
  2. Bei Flächen, die nur auf einer Seite texturiert sind, ist die andere Seite unsichtbar (Durchsichtigkeit auf 0 gesetzt). Das hilft dir, wenn du die falsche Seite einer Fläche texturiert hast. Benutze Kontextmenü → Flächen umkehren auf der Fläche, wenn das geschehen ist

Attribute

Dieses Plugin erweitert das Context-Menü um drei Einträge. Diese können benutzt werden, wenn du eine Fläche aktiviert hast:

Hard
Bestimmt, ob ein Flugzeug landet auf/oder zerschellt an dieser Fläche in X-Plane. Flächen "hard" zu machen ist "teuer" (framerate), also benutze es sparsam. Vertikale "hard" Flächen wirken sich nicht so aus, wie du es erwarten würdest; wenn du ein Gebäude für Kollisionen geeignet machen willst, benutze "hard" für das Dach, nicht für die Seitenflächen.
 
Ground
Bestimmt, ob eine Fläche mit "polygon offsetting" in X-Plane gerendert wird.
Benutze dies für Flächen, die flach auf dem Grund liegen, um Z-buffer thrashing in X-Plane zu verhindern. Benutze diesen Befehl nicht für andere Flächen.
 
Alpha
Dies ist ein Hinweis für X-Plane, dass diese Fläche mit einer teilweisen oder ganz transparenten oder durchscheinenden Textur versehen ist.
Benutze dies auf transparenten oder durchscheinenden Flächen, durch die du auf andere Flächen im selben Modell sehen willst. Drücke diesen Knopf nicht für opaque (normale) Flächen.

Export nach X-Plane

Wenn du es nicht schon getan hast, wähle Datei → Speichern unter… und sichere die .skp Datei in dem Aircraft- oder Scenery-Ordner, den du für deine X-Plane .obj Datei vorgesehen hast

Wenn du ein Terrain-Foto von Google Earth geladen hattest, musst du es vor dem Export ausblenden:

  • Wähle Fenster → Layer.
  • Im Dialog Layer vergewissere dich, dass "Google Earth Terrain" und "Google Earth Snapshot" beide abgewählt sind.
  • Wenn du das Terrain-Foto vor Beginn des Modellierens geladen hast, dann kannst du die Koordinaten, wo dein Modell in X-Plane platziert werden soll, in Dialog Fenster → Modellinformationen → Geostandort ablesen.

Dann:

  • Wähle Datei → Export X-Plane Object

Das Objekt wird im selben Ordner und mit demselben Namen wie die SketchUp Datei erzeugt, aber mit einer .obj Erweiterung.

Du köntest folgende Warnhinweise sehen:

Warning: You used multiple texture files. Using file name.
Du hast mehr als eine Textur-Datei benutzt, um die Flächen in seinem Modell zu texturieren.
X-Plane fordert, nur eine einzige Textur-Datei für alle deine Flächen zu benutzen.
 
Warning: n faces are untextured.
Du hast in deinem SketchUp Modell nicht auf alle Flächen Textur aufgetragen.
X-Plane wird untexturierte Flächen mit dem unteren linken Pixel deiner Textur-Datei färben (oder grau, wenn du keine Textur auf irgendeiner deiner Flächen platziert hast).

Wenn du eine der Warnungen siehst, wird X-Plane wahrscheinlich dein Modell nicht so darstellen, wie du es erwartet hast.

Einschränkungen

Konstruiere dein Sketchup Modell nur aus Rechtecken, Kreisen, Bögen, Komponenten und Gruppierungen - alle anderen Sketchup Typen werden nicht exportiert.

Dieses Plugin eignet sich nicht dazu, fortschrittliche X-Plane scenery primitives herzustellen. Das schließt ein:

  • Animationen
  • Attribute: z,B. Cockpit, Materialien etc
  • Level Of Detail
  • Lichter, Linien und Rauch
